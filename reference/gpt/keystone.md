# Keystone

## Overview

Keystone is a structured environmental observability and reconnaissance system designed to investigate, inventory, normalize, and analyze fragmented environmental remediation records across public regulatory repositories.

Keystone is not a chatbot project.

Keystone is an orchestration-first cognitive systems architecture focused on:
- environmental observability
- chronology reconstruction
- contradiction detection
- repository intelligence
- structured persistence
- longitudinal remediation analysis
- source-grounded environmental intelligence

The system currently focuses on Florida Superfund and major remediation sites but is designed for broader environmental observability applications.

---

# Core Philosophy

Keystone prioritizes:

- evidence-grounded retrieval
- structured persistence over conversational memory
- bounded cognition
- observability over narrative
- disciplined uncertainty handling
- source traceability
- modular orchestration
- efficiency through specialization

Keystone explicitly avoids:
- sensationalism
- unsupported conclusions
- activist framing
- speculative interpretation
- legal or policy conclusions
- emotional narrative generation

The system is designed to surface:
- what is documented
- what is unresolved
- what is inconsistent
- and what remains observable.

---

# Architectural Direction

Keystone is evolving toward a multi-agent orchestration architecture built around:
- small persistent runtime kernels
- structured JSON artifacts
- bounded retrieval phases
- model specialization
- transactional cognition
- persistence-first workflows

The long-term goal is:
high-quality environmental intelligence at minimal token and compute cost.

# Design Principles

## Structured Persistence Over Chat Memory

Chat history is transient orchestration only.

Durable cognition must persist as:
- JSON artifacts
- structured inventories
- chronology objects
- contradiction records
- monitoring datasets
- observability assessments

---

## Bounded Cognition

Keystone avoids:
- giant rolling contexts
- recursive context replay
- uncontrolled retrieval accumulation

Preferred workflow:

retrieve
→ extract
→ normalize
→ persist
→ unload
→ continue

---

## Tiered Cognition

Frontier reasoning models should only perform:
- ambiguity resolution
- contradiction adjudication
- observability reasoning
- chronology synthesis
- escalation decisions

Cheap models or deterministic systems should handle:
- cleanup
- parsing
- normalization
- extraction
- routing
- chunking
- deduplication

---

## Source Traceability

All meaningful findings should preserve:
- source repository
- source URL
- document references
- retrieval dates
- confidence state
- documented vs inferred distinction

---

## Observability Discipline

Keystone is not attempting to determine:
- whether contamination is morally bad
- whether institutions acted maliciously
- whether sites are "safe"

Keystone evaluates:
- observability quality
- continuity
- monitoring richness
- repository coherence
- contradiction state
- investigability

# Important Guidance For Future Agents

Future agents interacting with Keystone should:

- favor structured outputs over narration
- persist artifacts aggressively
- minimize rolling context growth
- avoid emotional framing
- preserve uncertainty honestly
- distinguish documented vs inferred findings
- prioritize observability over storytelling
- reduce repeated reasoning
- treat chat as transient
- treat artifacts as memory

Avoid:
- verbose chain-of-thought narration
- recursive summarization
- speculative interpretation
- uncontrolled retrieval loops
- duplicate document processing
- frontier-model use for low-value operations

The preferred Keystone pattern is:

small kernels
+ structured persistence
+ bounded phases
+ retrieval discipline
+ specialized cognition
+ orchestration efficiency
